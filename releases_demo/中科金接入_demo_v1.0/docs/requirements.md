# 需求基线：中科金智能平台对接

## 概述

在现有智能外呼中台中对接中科金智能平台的外呼能力，涉及四个模块，按依赖顺序迭代实现。原型为静态前端，Mock 数据驱动，无后端对接。

## 模块范围

| 期数 | 模块 | 类型 | 路由 key | 涉及文件 |
| --- | --- | --- | --- | --- |
| 1 | 业务场景维护 | 增量修改 | `sys-scene` | `js/pages/sys-scene.js`、`mock/data.js` |
| 2 | 外呼列表 | 增量修改 | `scene-list` | `js/pages/scene-list.js`、`mock/data.js` |
| 3 | 通话记录 | 新建页面 | `result-records` | `js/pages/result-records.js`、`mock/data.js`、`index.html`、`js/nav.js` |
| 4 | 通话统计 + 计费统计 | 新建页面 | `report-call`、`report-billing` | `js/pages/report-call.js`、`js/pages/report-billing.js`、`mock/data.js`、`index.html`、`js/nav.js` |

## 业务场景维护

- 在场景列表中新增「智能平台」列和筛选条件，支持四个平台：冰兰、科大讯飞、一知科技、中科金智能。
- 创建/编辑业务场景采用单页表单模式：智能平台选择（Radio）位于场景类型上方，平台相关字段随选择切换。
- 中科金平台字段（参照一知科技结构）：中科金场景 id（文本）、模型类型（Radio: 小模型/大模型）、选择中科金账号（Select，按模型类型级联过滤）。
- 四个平台的配置面板互斥展示（`onPlatformChange`）。
- 底部按钮统一为「取消」+「确定」。
- 删除场景时校验是否被运行中任务引用，若引用则拦截。

## 外呼列表

- 外呼任务卡片新增平台标签（一知科技/中科金智能），不同平台使用不同颜色标识。
- 筛选栏新增「智能平台」下拉筛选。
- 任务详情抽屉中展示平台信息。

## 通话记录

- 通话记录表格包含：序号、用户号码、通话开始/结束时间、通话时长、场景名称、通话状态、外呼总结、智能平台、最后通话节点、操作。
- 筛选栏支持按用户号码、场景名称、通话状态、智能平台过滤。
- 详情弹窗包含：音频播放卡片、对话转写记录、外呼小结（32 个信息字段）、外呼结果标签。

## 通话统计

- 双标签页：外呼统计（10 列）和客户统计（7 列）。
- 外呼统计维度：序号、场景名称、智能平台、外呼量、接通量、接通率、平均通话时长、意向客户数、意向率、通话总时长。
- 客户统计维度：序号、智能平台、新增客户数、外呼客户数、意向客户数、意向率、通话总时长。
- 支持按智能平台筛选。

## 计费统计

- 概览卡片：总消费金额、总通话次数、一知费用、中科金费用。
- 计费明细表格：序号、呼叫时间、场景名称、智能平台、通话时长、单价、通话次数、消费金额、模型类型。
- 支持按呼叫时间、智能平台、模型类型筛选。

## 交互参照

- 业务场景创建交互参照冰兰接入版 `sys-scene.js` 的单页表单模式。
- 中科金平台字段参照一知科技面板结构，不包含冰兰的呼叫通道、线路、时段、重拨等复杂字段。
- 外呼列表和通话记录的页面结构参照冰兰版对应文件。
- 所有页面 UI 风格与现有租户管理保持一致。

## 修改约束

- 不得重构与本次迭代无关的模块（租户管理、导航框架、公共工具）。
- 结构调整不得改变已有业务逻辑、Mock 语义、页面文案或交互结果。
- `index.html` 只用于入口结构、脚本引用和必要锚点。
- 新建页面使用 IIFE + `'use strict'`，注册到 `window.Pages`。
- 新建页面 JS 文件需在 `index.html` 中添加脚本引用，同时在 `js/nav.js` 中移除对应占位注册。
- 修改业务规则后同步更新 `memory/business-rules.md`。
- 作出新的结构或交互决策后同步更新 `docs/decisions.md`。
- 未确认的问题写入 `memory/open-items.md`，不得自行当作已确认需求。
