# 需求基线：意向标签管理

## 概述

在智能外呼中台系统管理中新增「标签管理」模块，用于统一管理供应商标签池、中台标准标签集，以及供应商标签到本地标签的映射关系。原型为静态前端，Mock 数据驱动，无后端对接。

## 模块范围

| 模块 | 类型 | 路由 key | 涉及文件 |
|------|------|----------|----------|
| 标签管理 | 新建页面 | `sys-tags` | `js/pages/sys-tags.js`、`mock/data.js`、`index.html`、`js/nav.js` |
| 业务场景 | 增量修改 | `sys-scene` | `js/pages/sys-scene.js`、`mock/data.js` |
| 线索记录 | 新建页面 | `result-clue` | `js/pages/result-clue.js`、`index.html`、`assets/css/app.css` |

## 标签管理

### 左侧配置树

- 双根节点：「中台标签集」和「供应商标签集」
- 供应商标签集三级结构：供应商 → 租户类型（门店/总部）→ 场景（督办/服务/回访/新线索/冷线索）
- 中台标签集两级结构：租户类型 → 场景
- 叶子节点（场景）显示绿色/灰色圆点指示器：绿色 = 有已启用标签配置
- 支持展开/折叠，选中节点高亮，自动展开父级路径

### 右侧标签配置（供应商模式）

- 标签表格包含：启用勾选、序号、本地编码、标签名称、映射本地标签下拉、排序、操作
- 排序支持行内编辑，修改后自动移位（shiftSortValues）
- 映射下拉选项来自中台标签集的对应租户类型+场景
- 工具栏：搜索（名称/编码）、状态下拉筛选、全选启用、清空启用
- 标签 CRUD：新增（Prompt 弹窗）、编辑名称和排序（行内编辑）、删除（确认弹窗 + 级联清理）

### 右侧标签配置（中台模式）

- 本地标准标签表格：序号、本地编码、标签名称、排序、操作
- 支持新增、编辑名称和排序、删除本地标签
- 删除前校验是否被供应商标签映射引用

### 弹窗

- 场景配置弹窗：CRUD + 启用/停用（600px 宽）
- 供应商管理弹窗：CRUD + 启用/停用（580px 宽）
- 通用 Prompt 弹窗：文本输入 + 重名校验
- 通用 Confirm 弹窗：删除确认

### 数据模型

- MockTagSuppliers（5个供应商：科大讯飞、冰兰-vcp、冰兰、一知科技、中科金）
- MockTenantTypes（2个租户类型：门店 store、总部 headquarters）
- MockTagScenes（5个场景：督办 db、服务 fw、回访 hf、新线索 xxs、冷线索 lxs）
- MockSupplierTagPool（各供应商标签池，共 ~45 个标签）
- MockTagConfigs（配置单元：key = supplierId_tenantType_sceneId，约 28 个配置）
- MockLocalTagSets（本地标准标签集：key = tenantType_sceneId，共 10 组）
- MockSupplierLocalTagMappings（供应商→本地映射表）

### 业务规则

- 排序移位：修改排序时自动调整同池/同集内其他标签，保持连续不重复
- 启用自动映射：新启用标签根据关键词自动推断映射到本地标签
- 停用保护：供应商或场景停用时，对应配置的操作均禁用
- 权限：所有功能仅限超级管理员；搜索和筛选不涉及权限控制
- 跨模块依赖：业务场景的智能平台和场景类型选项来源于标签管理数据

## 线索记录

- 筛选卡片（可折叠）：用户号码、最后回访时间、通话状态、场景名称 → 意向级别联动
- 数据表格（13列）：以线索维度展示回访统计
- 客户标签弹窗：6 维度标签（意向、关注、城市、到店、购车时间、通话标签）
- 回访详情弹窗：回访记录表 + 分页
- 通话详情抽屉：音频播放器 + 对话转写 + 外呼小结/详细信息 Tab

## 标注系统

- 通过 `data-anno` 锚点为业务元素添加可视化标注
- 标注点红色标记，可点击查看/编辑弹窗（10 个标准字段）
- 编辑保存到浏览器缓存，可选本地服务写回文件
- 当前标注：sys-tags 17 条 + sys-scene 3 条

## 修改约束

- 不得重构与当前迭代无关的模块
- 不得改变已有业务逻辑、Mock 数据语义、页面文案或交互结果
- `index.html` 只用于入口结构、脚本引用和必要锚点
- 新建页面使用 IIFE + `'use strict'`，注册到 `window.Pages`
- 新建页面 JS 需在 `index.html` 添加脚本引用，并在 `js/nav.js` 移除占位注册
- 修改业务规则后同步更新 `memory/business-rules.md`
- 作出新决策后同步更新 `docs/decisions.md`
