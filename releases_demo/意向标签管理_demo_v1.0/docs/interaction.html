7. 系统管理-租户管理-充值单配置-手动生效

1）功能名称： 充值单配置-手动生效

2）交互说明： 如图。

3）数据来源： 前端Mock数据。

4）逻辑说明： 关联充值单后有效期不再自动生成，需用户手动点击"生效"按钮触发有效期计算和激活。激活后同步更新历史充值记录中的有效期字段。

5）功能明细

功能名称功能说明数据来源取值逻辑数据格式默认值异常处理
有效期待生效状态充值单已支付但有效期尚未激活时，有效期列显示"待生效"前端Mock充值单状态=已支付 且 计费类型支持有效期（仅坐席费/坐席费+通话费）且 validityActivated=false文本未生成--
生效按钮-租户列表待生效状态下，租户列表操作列显示"生效"按钮前端行为hasPendingValidity=true 时渲染按钮隐藏--
生效按钮-充值单抽屉待生效状态下，充值单配置抽屉的有效期字段旁显示"生效"按钮前端行为hasPendingValidity=true 时渲染按钮隐藏--
手动生效触发点击"生效"按钮后自动计算并激活有效期前端JSvalidFrom：若存在历史有效期且未过期，从历史截止日期次日开始（无缝续期）；否则从关联日期开始。validTo = validFrom + 套餐天数 - 1天。套餐天数：半年套餐=180天，全年套餐=365天日期范围--充值单未支付提示"充值单未支付，无法生效"；已激活提示"有效期已生效，无需重复操作"
激活后状态更新激活成功后有效期从"待生效"变为计算后的日期范围，按钮消失，租户变为可呼叫状态前端JS设置 validityActivated=true、enabled=true，同步更新历史记录状态变更----
计费类型区分仅坐席费生成有效期；仅通话费增加话费不涉及有效期；坐席费+通话费同时支持前端Mock根据计费类型和套餐计算逻辑分支--仅通话费类型不显示"生效"按钮

五. 其他说明

1. 意向枚举值说明

智能平台租户类型数据来源取值逻辑意向等级枚举
一知总部百炼智能体取百炼提取的意向等级A (高意向)；B (潜在)；C (一般)；D (忙碌/敷衍)；E (拒绝/无效/无应答) ；F（未接通）
一知门店一知回调取一知返回的意向级别A (高意向)；B (低意向)；C (意向待定)；D (无意向)；E (未接通)；F（停机/空号）

百炼智能体：F（未接通）新增默认标签，如果客户未接通自动打上这个标签
