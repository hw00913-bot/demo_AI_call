一、标签管理（sys-tags）

权限范围：超级管理员（搜索和筛选不涉及权限控制）

1.1 左侧配置树

中台标签集

管理平台统一的标准标签库。按租户类型（门店租户/总部租户）和场景（督办/服务/回访/新线索/冷线索）组织本地标准标签，供应商标签最终需要映射到这些本地标签。

数据来源中台标签配置数据
取值逻辑每个租户类型+场景组合维护一组独立标准标签，标签编码按 LOCAL-{租户类型}-{场景编码}-{序号} 自动生成
字段序号、本地编码、本地标签名称、排序、操作（编辑/删除）

交互：点击展开租户类型节点，再点击场景节点加载对应标签列表。支持编辑标签名称、新增本地标签、删除本地标签。绿色圆点表示该场景已配置标签数量。

规则：同一租户类型+场景下，本地标签名称不可重复。删除前校验是否被供应商标签映射引用。场景停用时新增按钮禁用。

供应商标签集

管理各供应商（科大讯飞/冰兰/一知科技/中科金等）的标签池和启用配置。左侧按供应商→租户类型→场景三级结构组织。

数据来源供应商标签池配置数据
取值逻辑每个供应商维护独立标签池（编码按 TAG-{供应商编码}-{序号} 生成），通过 enabledTagIds 配置启用标签，供应商标签通过映射关联到中台标准标签
字段启用复选框、序号、本地编码、标签名称、映射本地标签下拉、排序、操作（编辑/保存/取消/删除）

交互：展开供应商→租户类型→场景节点，右侧加载标签配置表格。支持勾选启用/停用、编辑名称、删除、通过下拉映射本地标签。工具栏支持搜索、按启用状态筛选、全选启用/清空启用。

规则：标签名称同池内不可重复。删除前校验引用。供应商/场景停用时操作禁用。

1.2 工具栏

场景配置

管理标签配置中的场景列表（督办/服务/回访/新线索/冷线索）。支持新增、编辑、启用/停用和删除。

数据来源场景配置数据
入口工具栏「⚙ 场景配置」按钮 → 弹窗（600px）

交互：弹窗展示场景列表表格，支持行内编辑名称、切换启用/停用、删除。底部「+ 新增场景」创建新场景（Prompt 弹窗输入名称）。

规则：名称不可重复。删除前校验是否存在标签配置，有配置时拦截。

供应商管理

管理供应商列表。支持新增、编辑、启用/停用。每个供应商维护独立标签池。

数据来源供应商配置数据
入口工具栏「🏢 供应商管理」按钮 → 弹窗（580px）

交互：弹窗展示供应商列表（序号、编码、名称、标签数、状态、操作），支持行内编辑名称、切换状态。底部「+ 新增供应商」创建并初始化空标签池。

规则：名称不可重复。停用后对应配置中所有操作禁用。标签数为只读。

1.3 标签列表（供应商模式）

标签启用开关

勾选或取消勾选控制单个标签在当前场景配置中的启用状态。

数据来源场景配置的 enabledTagIds
取值逻辑勾选时标签ID添加到 enabledTagIds 并自动匹配映射；取消时移除

交互：点击首列复选框切换启用/停用。启用行正常显示，未启用行半透明。操作后自动更新计数和树节点指示器。

规则：供应商/场景停用时复选框禁用。

新增供应商标签

在当前供应商标签池中新增标签，默认未启用、排在末尾。

交互：点击「供应商标签池：新增标签」按钮 → Prompt 弹窗输入名称 → 校验重名 → 确认创建并刷新列表。

规则：名称不可重复、不能为空。供应商/场景停用时按钮禁用。

搜索标签

实时过滤标签列表，按名称或本地编码匹配（不区分大小写）。

交互：输入文字后实时过滤，清空恢复完整列表。与状态下拉筛选可叠加使用。

标签状态筛选

按启用状态筛选：全部标签 / 已启用 / 未启用。

交互：选择筛选项后列表实时刷新。

全选启用 / 清空启用

批量操作。

按钮行为
全选启用将当前供应商标签池所有标签ID写入 enabledTagIds，自动推断映射
清空启用将 enabledTagIds 置为空数组

规则：供应商/场景停用时按钮禁用。

映射本地标签

将供应商标签映射到中台本地标准标签，确保跨供应商语义一致。

数据来源供应商→本地映射表 + 中台本地标签集
取值逻辑下拉选项来自当前租户类型+场景的本地标签集。新启用标签根据关键词自动推断映射。手动选择优先于自动推断

交互：点击下拉选择目标本地标签（格式：本地编码 / 标签名称），选择后立即保存。映射数量实时更新。

规则：供应商/场景停用时下拉禁用。

标签操作（编辑/删除）

对单个标签进行编辑或删除。

交互：

  • 点击「✏ 编辑」→ 行内编辑：名称变为输入框、排序变为数字输入框 → 「✓ 保存」或「✕ 取消」
  • 点击「🗑 删除」→ 确认弹窗 → 级联删除（从标签池、所有配置、所有映射中移除）

规则：编辑时名称不能为空/重复，排序值 ≥1 的整数。删除时提示引用数量。供应商/场景停用时按钮禁用。

排序

控制标签展示顺序。编辑模式下可修改排序值，系统自动移位同标签池内其他标签。

数据来源供应商标签池各标签的 sort 字段
取值逻辑排序值 ≥1 整数。编辑触发 shiftSortValues() 移位：后移区间内 sort-1，前移区间内 sort+1。新增默认最大值+1

交互：点击编辑后排序列切换为数字输入框，保存时校验并移位，取消恢复原值。

规则：排序值非整数或 <1 时拒绝保存。

1.4 标签列表（中台模式）

新增本地标签

在当前租户类型+场景下新增中台标准标签，供应商映射下拉中可选用。

交互：点击「新增本地标签」→ Prompt 弹窗输入 → 校验重名 → 确认创建。

规则：名称不可重复、不能为空。场景停用时按钮禁用。

排序（中台模式）

与供应商模式排序共用 shiftSortValues() 函数，逻辑一致。新增本地标签默认排序为当前最大值+1。

1.5 弹窗

场景配置弹窗

弹窗形式的场景管理。打开后可拖拽移动、调整大小。表格内行内编辑、状态切换、删除。关闭后自动刷新左侧配置树。

供应商管理弹窗

弹窗形式的供应商管理。打开后可拖拽移动、调整大小。表格内行内编辑、状态切换。关闭后自动刷新左侧配置树。

二、业务场景（sys-scene）

权限范围：超级管理员

2.1 新增业务场景

打开右侧抽屉表单，创建新的外呼业务场景。

数据来源智能平台 → MockTagSuppliers（标签管理-供应商列表）；场景类型 → MockTagScenes（标签管理-场景列表）；可用租户 → MockTenantRows
字段场景名称、场景编码、场景描述、可用租户（多选）、智能平台、场景类型、数据导入方式、平台专属配置

交互：点击「+ 新增业务场景」按钮 → 右侧滑入抽屉 → 选择智能平台联动显示对应配置面板 → 填写字段 → 底部「确定」提交。

规则：智能平台和场景类型为必选项。平台切换时清空模型类型和场景传入信息。中科金平台默认预设「姓名/name」字段。

2.2 智能平台选择

选择业务场景所属的智能外呼平台。数据来源于标签管理模块的供应商列表（当前硬编码：冰兰、科大讯飞、一知科技、中科金智能）。

交互：点击 Radio 切换平台，联动显示对应平台配置面板（场景ID、模型类型、账号）。

规则：必选项。切换清空已选模型类型和场景传入信息。新增供应商时需同步更新此选项列表。

2.3 场景类型选择

选择业务场景的类型分类。数据来源于标签管理模块的场景列表（当前硬编码：首访、服务、回访、新线索、冷线索），其中「首访」对应标签管理中的「督办(db)」。

交互：点击 Radio 选择类型。

规则：必选项。新增场景时需同步更新此选项列表。
