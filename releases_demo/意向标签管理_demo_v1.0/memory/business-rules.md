# 业务规则

## 标签排序移位规则

修改标签排序值时，系统自动调整同标签池（供应商模式）或同标签集（中台模式）内其他标签的排序，保持排序值连续不重复。

- **后移**（newSort > oldSort）：排序值在 `(oldSort, newSort]` 区间内的其他标签，sort 减 1
- **前移**（newSort < oldSort）：排序值在 `[newSort, oldSort)` 区间内的其他标签，sort 加 1
- **不变**（newSort = oldSort）：无操作

示例：A=1, B=2, C=3, D=4，编辑 C 的排序：
- C: 3→1 → A=2, B=3, C=1, D=4
- C: 3→4 → A=1, B=2, C=4, D=3

## 标签启用自动映射

供应商标签首次启用时，系统根据标签名称关键词自动推断映射到本地标准标签：
- 高/强烈/S级/A级/近期成交 → 高意向/高优先级类
- 中/B级/C级/考虑/潜在 → 中意向/需跟进类
- 低/D级/E级/一般 → 低意向/门店处理类
- 无效/空号/停机/异常 → 无效客户/服务预警类

## 停用保护

供应商或场景状态为 `disabled` 时，对应配置节点下的启用勾选、编辑、删除、新增操作均禁用。

## 权限范围

标签管理模块所有功能仅限超级管理员使用。搜索和状态筛选属于前端过滤，不涉及权限控制。

## 跨模块数据依赖

### 业务场景 → 标签管理

业务场景管理（sys-scene）的新增业务场景表单中，以下字段的数据来源于标签管理模块：

| 字段 | 数据来源 | 对应关系 |
|------|---------|---------|
| 智能平台 | `MockTagSuppliers`（供应商列表） | 冰兰、科大讯飞、一知科技、中科金智能 |
| 场景类型 | `MockTagScenes`（场景列表） | 首访→督办(db)、服务→服务(fw)、回访→回访(hf)、新线索→新线索(xxs)、冷线索→冷线索(lxs) |

**约束**：
- 在标签管理中新增供应商时，需同步更新业务场景的智能平台选项
- 在标签管理中新增场景时，需同步更新业务场景的场景类型选项
- 当前两处均为硬编码，未来应改为从 MockTagSuppliers / MockTagScenes 动态读取

## 线索记录

### 数据筛选

线索记录页面支持四重筛选条件，可组合使用：
- 用户号码：模糊匹配（indexOf）
- 最后通话状态：精确匹配
- 场景名称：精确匹配
- 意向级别：精确匹配（仅在选择了场景名称后显示，选项从该场景的数据中动态提取）

### 客户详细标签

每条线索记录包含 6 个维度的客户标签：
- 意向标签（如 C(待筛选)、B(无购车计划)、O(同意加微信)、H(客户已买车)）
- 用户关注（如价格优惠、金融优惠、预约到店等）
- 购车城市
- 预计到店时间
- 预计购车时间
- 通话标签

### 意向级别联动

选择场景名称后，意向级别下拉选项动态从该场景已有的线索数据中提取唯一值，避免显示其他场景的意向级别。

### 通话详情抽屉

从回访详情中「详情」链接进入，展示：
- 左侧：音频播放器 + 对话转写记录（5轮对话）
- 右侧：外呼小结 / 详细信息 Tab 切换
  - 外呼小结：意向等级、计划到店时间、预计购车时间、意向品牌/车系等
  - 详细信息：会话ID、用户号码、通话状态、通话时间、场景名称等
