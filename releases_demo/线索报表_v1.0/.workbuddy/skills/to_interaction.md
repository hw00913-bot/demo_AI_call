# Skill: 生成功能说明文档

**说明**：  
此 AI Skill 用于直接生成结构化的 Markdown 功能说明文档（`功能说明文档.md`），遵循 5 段固定结构，以表格形式呈现功能明细。无需先读 HTML 等源码再二次转换——AI 直接从项目理解中产出 MD 文档即可。

## 触发条件
当用户说"生成说明文档"、"调用 to_interaction.md"、"生成功能说明"或类似需求时，触发此技能。

## 规范约束：功能说明文档结构
当负责撰写或更新《功能说明文档》时，整体内容必须严格遵从以下 5 个固定结构：
1. **版本说明**：【注意】只有当用户明确要求生成此项时才创建。若创建，必须使用**表格方式**显示，且表头严格为：`版本号` | `更新时间` | `更新内容`。
2. **项目范围**：必须使用**表格方式**显示，说明项目的功能范围。表头严格为：`系统名称` | `涉及板块` | `功能范围`。
3. **逻辑说明**：必须使用**表格方式**显示，表头严格为：`流程名称` | `跳转地址`。用于描述功能的关联流程及链接跳转。
4. **功能说明**：作为一级标题"四. 功能说明"。内部必须严格遵从以下多级递增标号的结构（若有多个板块和功能，须依次生成标号）：
   - 二级标题：`1. 板块名称`（例：1. 板块A、2. 板块B）
   - 三级标题：
     - `1）功能名称`
     - `2）交互说明`
     - `3）数据来源`
     - `4）逻辑说明`
     - `5）功能明细`：此部分必须使用**表格方式**显示，表头严格为：`功能名称` | `功能说明` | `数据来源` | `取值逻辑` | `数据格式` | `默认值` | `异常处理`。
5. **其他说明**：作为一级标题"五. 其他说明"。用于放置如"状态枚举映射表"等全局、通用的表格及说明内容。

## 执行步骤

1. **直接生成 Markdown 文档**：按上述 5 段结构，生成 `功能说明文档.md`。无需生成 CSV 或中间文件。
2. **生成方式**：使用 Markdown 表格（`| --- | --- |` 语法）+ 加粗三级标题（`**1）功能名称**`），保证文档在 GitHub 或本地预览器中可读。
3. **默认输出**：目标路径默认为 `功能说明文档.md`（项目根目录）。生成完成后通知用户即可。
