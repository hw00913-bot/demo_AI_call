## 智能外呼中台 - 业务场景三步向导 + select 高度问题修复

### 三步向导实现（DeepSeek + 后续修复）

**核心文件：** `js/pages/sys-scene.js`、`style.css`

**步骤切换机制：** `goToStep(step)` 函数通过 inline `style.display` 控制三个 `.step-content` 区域的显隐，步骤条状态用 `active`/`completed` 类控制，底部按钮根据步骤动态渲染。

**Step 2 表单字段：** 智能平台（radio，冰兰/科大讯飞/一知科技）、数据导入方式（radio）、一知科技场景id（input）、模型类型（radio，小模型/大模型）、选择一知账号（select，仅大模型时显示）。

### select 高度 36px 问题修复

**问题原因：** `<select>` 元素的 `height` 在浏览器中会被 UA 默认样式、内部 padding、line-height 覆盖，`.biz-form-input` 的 34px 无法被覆盖导致 select 偏矮。

**解决方案：**
- 新增独立 CSS 类 `.biz-form-select`，完整重写 select 所有属性：
  - `height: 36px`、`line-height: 36px`（文字垂直居中）
  - `box-sizing: border-box`、`appearance: none`
  - 自定义 SVG 下拉箭头（`background-image`）
- HTML 上将 `class="biz-form-input"` 改为 `class="biz-form-select"`，与 input 的 34px 完全解耦

**同时修复：** `onModelTypeChange()` 从 `row.style.display` 改为 `row.classList.toggle('hidden', !checked)`，避免重置 `display:''` 时丢失行内 `align-items:flex-start` 样式。

### 踩坑经验

- **select 高度问题：** 纯设 `height` + `!important` 对 `<select>` 无效，必须用独立类完整重写所有相关属性（`line-height`、`padding`、`box-sizing`、`appearance`），并自定义下拉箭头去掉浏览器默认样式干扰。
