# 已确认业务规则

## 租户计费

- 租户计费管理入口位于系统管理的租户管理列表。
- 租户列表支持导出全部租户的计费汇总，导出口径必须复用页面实时汇总结果。
- 导出扣费总额 = 租户实际消费金额 + 已生效手工扣减金额；冻结总和按有效冻结分钟数乘任务单价快照汇总。
- 导出可用余额 = max（历史充值总额 - 扣费总额 - 冻结总和，0），大模型和小模型可用分钟数分别按各自默认单价换算。
- 当前版本每个租户只按模型类型配置大模型默认价和小模型默认价。
- 供应商是未来可选覆盖维度，当前不展示、不配置、不参与价格选择。
- 未来供应商专属价的匹配优先级为：租户 + 模型 + 供应商专属价，其次回退租户 + 模型默认价。
- 充值单号通过输入框读取校验，读取后反显门店编码、门店名称和计费信息。
- 关联类型包括试用单和付费单，默认付费单；付费单沿用外部充值单读取与二次确认规则。
- 试用单由运营选择有效时间、填写通话费用，两者至少填写一项；不读取外部充值单号。
- 试用有效月份按每月 30 日换算为 30、60、90 日，仅填写通话费用时周期为 0。
- 试用单按实际填写内容确定计费类型：仅坐席费、仅通话费或坐席费加通话费，关联后先进入待生效。
- 读取充值单只生成预览，不写入历史关联记录；用户点击“确认关联”后才正式关联。
- 充值单所属租户必须与当前租户匹配，门店不匹配时禁止确认关联。
- 仅状态为“已支付”的充值单可关联，并且充值单号在历史关联记录中必须唯一。
- 充值单关联后默认待生效，不立即计入有效期或统一资金余额。
- 生效操作属于单条关联充值单，不允许按租户批量或模糊生效。
- 不存在、查询失败、已取消等异常状态需要保留，但不得加入历史关联充值单。
- 历史关联记录展示关联类型，不提供删除按钮；付费单校验已支付，试用单无需外部支付状态校验。

## 套餐与余额

- 计费类型包括仅坐席费、仅通话费、坐席费加通话费。
- 坐席费套餐当前包括半年套餐和全年套餐；默认添加时长分别为 183 日和 365 日。
- 含坐席费的充值单点击生效后先弹出有效时长确认弹窗，反显套餐默认天数并允许调整；调整值必须为大于 0 的整数。
- 仅坐席费充值单确认时长后更新有效期。
- 仅通话费充值单点击生效后先弹出添加通话余额确认弹窗，按充值单反显金额并允许调整，确认后计入统一资金余额。
- 坐席费加通话费充值单点击生效后先弹出有效期和通话余额联合确认弹窗，允许同时调整添加天数和添加金额，确认后同时更新有效期和余额。
- 只有已生效的通话充值金额才进入租户统一资金账户。
- 外部充值单只提供充值金额，不区分大模型或小模型。
- 所有已支付通话充值进入租户统一资金账户，扣除实际消费金额后形成资金余额。
- 大模型和小模型通话余额是同一资金余额按对应模型默认价换算出的分钟视图，不能相加。
- 冻结任务区分模型类型和供应商，冻结金额等于冻结分钟数乘以任务创建时的单价快照。
- 可用金额等于资金余额减全部冻结任务金额之和。
- 大模型和小模型可用分钟数均由同一可用金额除以对应模型默认价得到。
- 未来启用供应商专属价时，调价只影响后续任务，不追溯已冻结或已扣费任务。
- 资金余额等于已生效通话充值总额减手工扣减金额和已消费金额。
- 余额调整仅支持手工扣减，通过余额调整 Tab 的”手工调整”按钮弹窗操作，不关联充值单、不提供手工补回。
- 手工扣减只校验当前资金余额，不减去冻结金额，也不修改已有冻结任务。
- 扣减后资金余额低于有效冻结金额时，已有冻结保持不变，可用金额按最低 0 计算；不设置独立的“冻结覆盖不足”全局拦截状态。
- 可用余额作为内部隐藏字段参与计算和刷新，不在页面以金额形式展示。
- 总部类型租户走与门店租户相同的计算逻辑，不设特殊代码分支；通过配置超大额充值单实现几乎无限的可用分钟数，充值单有效期 2026-01-01 ~ 2099-12-31。
- 已生效调整流水不得删除或修改。

## 冻结与扣费

- 发起外呼时按任务模型类型和供应商冻结预计分钟数，并使用单价快照换算冻结金额。
- 每次导入所需冻结金额 = 号码数 × 单号码预计分钟数 × 解析后的有效单价。
- 本次所需冻结金额不超过当前可用金额时允许导入；超出时拒绝并提示最大可导入号码数。
- 最大可导入号码数 = floor（当前可用金额 ÷ 单号码预计冻结金额）。
- 最终根据实际通话分钟数和单价快照计算实际扣费金额。
- 未使用的冻结金额需要释放回统一资金账户。
- 任务已完成或已终止时立即释放对应冻结占用。
- 冻结中记录从创建时间起满 24 小时后自动释放。
- 释放记录需要保存释放时间和释放原因，并保证幂等。
- 释放仅减少冻结总额，不直接增加资金余额；实际消费由结算单独扣减。
- 原型使用冻结中的外呼任务 Mock 数据模拟冻结和释放。
- 完整扣减链路由流程图表达，不在当前原型中实现后端账务逻辑。

## 呼叫控制

- 页面展示“呼叫控制状态”，状态包括可发起和不可发起。
- 租户在有效期内且统一可用金额大于 0 时，满足基础可呼叫条件。
- 手动控制状态与基础条件共同决定最终呼叫控制状态。
- 过期或可用分钟数为 0 时，管理呼叫按钮置灰。
- 统一可用金额为 0 时管理呼叫按钮置灰；可用金额大于 0 时仍需按本次导入规模执行冻结校验。
