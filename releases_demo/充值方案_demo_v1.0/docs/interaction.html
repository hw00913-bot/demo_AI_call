0. 系统管理-租户管理-计费汇总导出

1）功能名称： 租户计费汇总导出

2）交互说明： 点击租户列表右上角“导出”按钮，直接下载当前全部租户的计费汇总 CSV 文件。

3）数据来源： 租户基础信息及 getTenantBillingSummary 实时汇总结果。

4）逻辑说明： 扣费总额 = 实际消费金额 + 已生效手工扣减金额；冻结总和为当前有效冻结金额；可用余额 = max（历史充值总额 - 扣费总额 - 冻结总和，0）。大小模型可用分钟数分别按对应默认单价换算。

5）功能明细

功能名称功能说明数据来源取值逻辑数据格式默认值异常处理
导出范围导出当前全部租户MockTenantRows逐租户调用getTenantBillingSummaryCSV全部租户无租户时导出仅含表头
导出字段租户名称、租户ID、租户类型、历史充值总额、扣费总额、冻结总和、可用余额、大小模型可用分钟数、有效期、呼叫控制状态基础信息+计费汇总按页面当前计费口径实时计算文本/数值金额和分钟保留2位模型未配置时输出“未配置”
导出文件文件名为“租户计费汇总_YYYY-MM-DD.csv”前端JSUTF-8 BOM编码CSV文件当天日期--

1. 系统管理-租户管理-充值管理

1）功能名称： 充值管理抽屉（三Tab：计费明细 / 充值单管理 / 余额调整）

2）交互说明： 点击租户列表"充值管理"按钮打开右侧抽屉。租户信息区展示租户名称、有效期、历史充值总额和呼叫控制状态（可用余额为内部隐藏字段，不对外展示）。充值单管理先选择关联类型：付费单输入单号后读取并反显门店信息，核对后确认关联；试用单选择1/2/3个月有效时间并填写通话费用后确认关联。三个 Tab 共用租户信息区。

3）数据来源： 前端Mock数据（MockTenantBillingRows、MockTenantRechargeHistory、MockTenantPriceRules、MockTenantFrozenTasks、MockTenantBalanceAdjustments）。

4）逻辑说明： 资金余额 = 已生效通话充值总额 - 手工扣减金额 - 已消费金额。冻结总额 = 有效冻结任务的冻结分钟数 × 单价快照。可用余额 = max(资金余额 - 冻结总额, 0)。每分钟数按对应模型默认单价换算。总部类型租户无特殊代码逻辑，通过配置超大额充值单实现几乎无限的可用分钟数。

3. 外呼场景-外呼列表-手动导入-冻结校验

1）功能名称： 导入时冻结金额校验

2）交互说明： 手动导入弹窗"上传文件"Tab展示本次冻结校验信息：租户、模型/供应商、计费单价及来源、单号码预计分钟、最大可导入数、本次预计冻结金额和校验结果。调整解析号码数时实时更新校验结果。

3）数据来源： 场景Mock数据中的计费上下文（tenantName/modelType/vendorCode/vendorName/estimatedMinutesPerPhone）+ 租户计费数据。

4）逻辑说明： freezeAmountPerPhone = estimatedMinutesPerPhone × unitPrice（按租户+模型+供应商解析有效单价，供应商专属价优先于模型默认价）。maxImportCount = floor(availableAmount / freezeAmountPerPhone)。号码数×单号码预计冻结金额≤可用金额时允许导入，超出时拒绝并展示最大可导入数。

4. 外呼场景-外呼列表-冻结任务生命周期

1）功能名称： 冻结任务自动释放与场景终止联动释放

2）交互说明： 冻结任务在以下情况自动释放：任务状态为已完成/已终止时；冻结创建超过24小时时。场景列表执行"终止"操作时联动释放该场景下的有效冻结。

3）数据来源： 前端Mock数据（MockTenantFrozenTasks）。

4）逻辑说明： 每次计算计费汇总前先执行syncFrozenTaskReleases()自动同步释放。释放条件：status=冻结中 且 (taskStatus=已完成 或 taskStatus=已终止 或 createdAt距今≥24小时)。释放操作：status→已释放，记录releasedAt和releaseReason，幂等保证不重复释放。场景终止时额外调用releaseFrozenTasksByScene按场景名批量释放。

5）功能明细

功能名称功能说明数据来源取值逻辑数据格式默认值异常处理
自动释放每次渲染时自动检测并释放符合条件的冻结任务前端JSsyncFrozenTaskReleases遍历MockTenantFrozenTasks后台逻辑--无效createdAt视为已过期
释放原因-任务完成taskStatus=已完成时释放前端JSgetFrozenReleaseReason返回"任务已完成"文本----
释放原因-任务终止taskStatus=已终止时释放前端JSgetFrozenReleaseReason返回"任务已终止"文本----
释放原因-超时创建时间距今≥24小时时释放前端JSgetFrozenReleaseReason返回"冻结超过24小时"文本----
场景终止联动外呼列表"终止"操作时调用releaseFrozenTasksByScene批量释放该场景冻结前端JS按sceneName匹配，status=冻结中时释放，返回释放笔数和金额联动调用--幂等，不重复释放
释放记录释放时写入releasedAt（释放时间）和releaseReason（释放原因）前端JSreleaseFrozenTask更新字段字段更新空已释放状态跳过
