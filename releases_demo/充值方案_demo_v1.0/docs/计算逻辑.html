1. 核心原则

  • 外部充值只提供金额，不区分大模型、小模型或供应商。
  • 租户使用统一资金账户。
  • 大模型和小模型分钟数是同一资金余额按不同单价换算的结果，不能相加。
  • 余额调整仅支持手工扣减，不关联充值单，不提供手工补回。
  • 可用余额作为内部隐藏字段参与计算，页面不直接展示金额。
  • 冻结任务使用任务创建时的单价快照，后续调价不影响已有任务。

2. 变量定义

变量含义来源
\(R\)已生效的通话充值总额历史充值单（已支付+已生效+含通话费）
\(D\)累计手工扣减金额余额调整流水（MANUAL_DEDUCT / OUT）
\(C\)累计实际通话消费金额租户主数据 consumedAmount
\(B\)当前资金余额\(B = R - D - C\)
\(F_i\)第 \(i\) 个任务的冻结金额\(F_i = m_i \times p_i\)
\(F\)当前有效冻结总额\(F = \sum F_i\)（仅含冻结中且未过期任务）
\(A\)当前可用余额（隐藏字段）\(A = \max(B - F, 0)\)
\(m_i\)第 \(i\) 个任务的预计通话分钟数任务创建时指定
\(m_{a,i}\)第 \(i\) 个任务的实际通话分钟数通话完成后统计
\(p_i\)第 \(i\) 个任务的单价快照任务创建时按价格优先级解析后锁定
\(P_L\)大模型默认单价MockTenantPriceRules（MODEL_DEFAULT）
\(P_S\)小模型默认单价MockTenantPriceRules（MODEL_DEFAULT）
\(P(t,m,v)\)按价格优先级解析的最终单价供应商专属价优先，模型默认价兜底

3. 充值金额入账

只有已支付且已生效的以下充值单计入统一资金账户：

  • 仅通话费 — 充值金额计入资金余额
  • 坐席费+通话费 — 通话费部分计入资金余额，坐席费部分更新有效期

仅坐席费充值单只更新租户有效期，不增加通话资金。

\[ R = \sum_{j=1}^{n} R_j \]

其中 \(R_j\) 是第 \(j\) 笔已生效通话充值金额。

4. 资金余额 核心公式

\[ \boxed{B = R - D - C} \]

即：资金余额 = 已生效通话充值总额 − 累计手工扣减 − 累计实际通话消费

5. 手工扣减

设本次手工扣减金额为 \(d\)，允许扣减的条件为：

\[ \boxed{0 < d \le B} \]

手工扣减只校验当前资金余额，不减去冻结金额，因此不要求 \(d \le B - F\)。

扣减完成后：

\[ B' = B - d \qquad D' = D + d \]

手工扣减规则

规则说明
不关联充值单扣减与具体充值单无关，不校验单笔充值剩余可退金额
不占用冻结扣减不修改或释放已有冻结任务，冻结任务保持不变
不提供补回不支持手工补回、撤销或修改已生效的扣减流水
必填原因扣减金额和扣减原因必须填写
流水不可删扣减流水生效后不可删除

6. 供应商独立单价

6.1 价格配置层级

  1. 租户 + 模型默认价（当前版本已实现）
  2. 租户 + 模型 + 供应商专属价（未来扩展，未配置时回退模型默认价）

6.2 价格解析优先级

最终单价按照供应商专属价优先、模型默认价兜底解析：

\[ \boxed{ P(t,m,v)= \begin{cases} P_{\text{vendor}}(t,m,v), & \text{存在启用的供应商专属价}\\ P_{\text{default}}(t,m), & \text{否则} \end{cases} } \]

示例

租户 A 大模型默认价 0.40 元/分钟，科大讯飞专属价 0.45 元/分钟：

  • 使用科大讯飞 → 0.45 元/分钟
  • 使用其他供应商 → 0.40 元/分钟

6.3 单价快照

任务创建时锁定单价快照：

\[ \boxed{p_i = P(t_i, m_i, v_i)} \]

后续调价不影响已创建任务的冻结和结算。

7. 任务冻结

\[ \boxed{F_i = m_i \times p_i} \]

租户有效冻结总额：

\[ \boxed{F = \sum_{i=1}^{n} F_i} \]

有效冻结条件

第 \(i\) 个任务计入有效冻结总额，必须同时满足：

\[ \boxed{ \text{冻结状态}_i = \text{冻结中} \;\land\; \text{任务状态}_i \notin \{\text{已完成}, \text{已终止}\} \;\land\; T_{\text{now}} - T_{\text{created},i} < 24\text{小时} } \]

任务已完成、已终止或冻结创建满 24 小时时，立即解除冻结占用。释放记录保存释放时间和释放原因，并保证幂等。

即使后续修改模型默认价或供应商专属价，已有任务的冻结金额也不会改变。

8. 可用余额 核心公式

\[ \boxed{A = \max(B - F,\; 0)} \]

可用余额只作为内部隐藏字段，用于：

  • 换算大模型可用分钟数
  • 换算小模型可用分钟数
  • 校验本次导入所需冻结金额

当 \(B < F\) 时，\(A = 0\)，最大可导入数自然为 0，不再设置独立的全局拦截状态。

9. 本次导入冻结校验

设 \(N\) 为本次导入号码数，\(Q\) 为单号码预计通话分钟数：

\[ \boxed{F_{\text{request}} = N \times Q \times P(t, m, v)} \]

允许导入的条件：

\[ \boxed{F_{\text{request}} \le A} \]

最大可导入号码数：

\[ \boxed{ N_{\max} = \left\lfloor \frac{A}{Q \times P(t, m, v)} \right\rfloor } \]

处理规则

  • 仍有效的已有冻结任务保持不变。
  • 已完成、已终止或超过 24 小时的冻结任务先自动释放，再计算可用金额。
  • 本次导入数不超过 \(N_{\max}\) 时允许导入，并新增对应冻结记录。
  • 本次导入数超过 \(N_{\max}\) 时拒绝导入，并提示最大可导入数。

10. 模型分钟数换算

10.1 通话余额分钟数

\[ \boxed{M_{LB} = \frac{\max(B, 0)}{P_L}} \qquad \boxed{M_{SB} = \frac{\max(B, 0)}{P_S}} \]

10.2 可用分钟数

\[ \boxed{M_{LA} = \frac{A}{P_L}} \qquad \boxed{M_{SA} = \frac{A}{P_S}} \]

⚠ 大小模型分钟数是同一金额池的两种换算视图，\(M_{LA} + M_{SA}\) 没有业务含义，禁止相加。

10.3 供应商专属价视图（未来扩展）

启用供应商专属价后，可用分钟数按供应商单价换算：

\[ \boxed{ M_A(t, m, v) = \frac{A}{P(t, m, v)} } \]

不同模型、不同供应商的分钟数都是同一可用余额的不同换算视图，不是多个独立余额，均不可相加。

11. 实际通话结算

11.1 结算计算

第 \(i\) 个任务的实际消费金额：

\[ \boxed{C_i = m_{a,i} \times p_i} \]

未使用的冻结金额（释放回资金池）：

\[ \boxed{U_i = \max(F_i - C_i,\; 0)} \]

任务结算后：

\[ B' = B - C_i \qquad F' = F - F_i \qquad A' = \max(B' - F',\; 0) \]

11.2 自动释放条件

设冻结创建时间为 \(T_c\)，当前时间为 \(T_n\)，任务状态为 \(S\)：

\[ \boxed{ S \in \{\text{已完成}, \text{已终止}\} \quad\lor\quad T_n - T_c \ge 24\text{小时} } \]

释放后该任务不再计入有效冻结总额（\(F_i' = 0\)）。资金余额 \(B\) 不因释放直接变化；实际消费由结算动作单独扣减。

释放原因判定优先级

优先级条件释放原因
1taskStatus = 已完成任务已完成
2taskStatus = 已终止任务已终止
3createdAt 距今 ≥ 24h冻结超过24小时

12. 呼叫条件

租户允许发起新呼叫任务，需要同时满足：

\[ \boxed{ \text{租户在有效期内} \;\land\; A > 0 \;\land\; \text{手动呼叫控制已启用} } \]

创建具体任务时，还必须解析出该任务的有效单价：

\[ \boxed{P(t, m, v) > 0} \]

禁止发起呼叫的情况

条件说明
不在有效期内租户有效期已过或未生成
手动已停用管理员手动关闭了该租户的呼叫开关
无有效单价模型默认价和供应商专属价均不存在或已停用
冻结超限本次预计冻结金额 > 可用金额

13. 综合公式 一图看懂

\[ \boxed{ A = \max(\; R - D - C - F,\; 0 \;) } \]

可用金额 = max( 已生效通话充值 − 手工扣减 − 实际消费 − 有效冻结金额, 0 )

数据流图： 
充值单(已支付) → [逐单生效] → R(充值总额)
                                    ↘
                              B = R − D − C  (资金余额)
                                    ↘
                              A = max(B − F, 0)  (可用余额，隐藏)
                               ↙              ↘
              M_LA = A / P_L              M_SA = A / P_S
           (大模型可用分钟数)          (小模型可用分钟数)
                               ↘
                      F_request = N × Q × P(t,m,v)
                      (导入时校验：≤ A 才允许)

14. 计算示例

示例数据

参数值
已生效通话充值总额 \(R\)2000 元
累计手工扣减 \(D\)100 元
累计实际消费 \(C\)180 元
有效冻结总额 \(F\)720 元
大模型单价 \(P_L\)0.40 元/分钟
小模型单价 \(P_S\)0.26 元/分钟

计算结果

资金余额\(B = 2000 - 100 - 180 = \mathbf{1720}\) 元
可用余额\(A = \max(1720 - 720, 0) = \mathbf{1000}\) 元
大模型通话余额\(M_{LB} = 1720 / 0.40 = \mathbf{4300}\) 分钟
大模型可用分钟\(M_{LA} = 1000 / 0.40 = \mathbf{2500}\) 分钟
小模型通话余额\(M_{SB} = 1720 / 0.26 \approx \mathbf{6615.38}\) 分钟
小模型可用分钟\(M_{SA} = 1000 / 0.26 \approx \mathbf{3846.15}\) 分钟

场景：再扣减 1100 元

\(B' = 1720 - 1100 = 620\)，由于 \(620 < 720\)，因此 \(A' = 0\)。

大模型和小模型可用分钟数均为 0，本次最大可导入号码数也为 0。该结果来自请求级冻结校验，不再使用独立的全局状态。

场景：导入校验

可用金额 \(A = 1000\)，大模型默认价 0.40 元/分钟，单号码预计 1 分钟：

\[ N_{\max} = \left\lfloor \frac{1000}{1 \times 0.40} \right\rfloor = \mathbf{2500} \text{ 条} \]
导入数预计冻结结果
2000 条2000 × 1 × 0.40 = 800 ≤ 1000✅ 允许导入
5000 条5000 × 1 × 0.40 = 2000 > 1000❌ 拒绝，提示最多 2500 条
