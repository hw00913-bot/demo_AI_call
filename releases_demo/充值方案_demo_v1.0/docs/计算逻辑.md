# 充值计费计算逻辑

## 1. 核心原则

- 外部充值只提供金额，不区分大模型、小模型或供应商。
- 租户使用统一资金账户。
- 大模型和小模型分钟数是同一资金余额按不同单价换算的结果，不能相加。
- 余额调整仅支持手工扣减，不关联充值单，不提供手工补回。
- 可用余额作为内部隐藏字段参与计算，页面不直接展示金额。
- 冻结任务使用任务创建时的单价快照，后续调价不影响已有任务。

## 2. 变量定义

| 变量 | 含义 |
| --- | --- |
| \(R\) | 已生效的通话充值总额 |
| \(D\) | 累计手工扣减金额 |
| \(C\) | 累计实际通话消费金额 |
| \(B\) | 当前资金余额 |
| \(F_i\) | 第 \(i\) 个任务的冻结金额 |
| \(F\) | 当前有效冻结总额 |
| \(A\) | 当前可用余额，内部隐藏字段 |
| \(m_i\) | 第 \(i\) 个任务的预计通话分钟数 |
| \(m_{a,i}\) | 第 \(i\) 个任务的实际通话分钟数 |
| \(p_i\) | 第 \(i\) 个任务创建时的单价快照 |
| \(P_L\) | 当前大模型默认单价 |
| \(P_S\) | 当前小模型默认单价 |
| \(t\) | 租户 |
| \(m\) | 模型类型，取值为大模型或小模型 |
| \(v\) | 供应商 |
| \(P_{\text{default}}(t,m)\) | 租户在指定模型下的默认单价 |
| \(P_{\text{vendor}}(t,m,v)\) | 租户、模型和供应商对应的专属单价 |
| \(P(t,m,v)\) | 按价格优先级解析出的最终单价 |

## 3. 充值金额入账

只有已支付且已生效的以下充值单计入统一资金账户：

- 仅通话费
- 坐席费加通话费

仅坐席费充值单只更新租户有效期，不增加通话资金。

通话充值总额为：

\[
R=\sum_{j=1}^{n}R_j
\]

其中，\(R_j\) 是第 \(j\) 笔已生效通话充值金额。

## 4. 资金余额

资金余额计算公式：

\[
\boxed{B=R-D-C}
\]

即：

\[
\text{资金余额}
=
\text{已生效通话充值总额}
-
\text{累计手工扣减}
-
\text{累计实际通话消费}
\]

## 5. 手工扣减

设本次手工扣减金额为 \(d\)，允许扣减的条件为：

\[
\boxed{0<d\le B}
\]

手工扣减只校验当前资金余额，不减去冻结金额，因此不要求：

\[
d\le B-F
\]

扣减完成后：

\[
B'=B-d
\]

\[
D'=D+d
\]

手工扣减规则：

- 不关联具体充值单。
- 必须填写扣减金额和扣减原因。
- 不修改或释放已有冻结任务。
- 不提供手工补回、撤销或修改。
- 扣减流水生效后不可删除。

## 6. 供应商独立单价

### 6.1 价格配置层级

价格配置支持以下两个层级：

1. 租户加模型默认价。
2. 租户加模型加供应商专属价。

当前版本只要求配置租户的大模型默认价和小模型默认价。供应商专属价是未来可选扩展，未配置时自动使用模型默认价。

### 6.2 价格解析优先级

最终单价按照“供应商专属价优先、模型默认价兜底”解析：

\[
\boxed{
P(t,m,v)=
\begin{cases}
P_{\text{vendor}}(t,m,v), & \text{存在启用的供应商专属价}\\
P_{\text{default}}(t,m), & \text{否则}
\end{cases}
}
\]

例如：

- 租户 A 的大模型默认价为 0.40 元/分钟。
- 租户 A 的大模型在供应商“科大讯飞”下配置专属价 0.45 元/分钟。
- 使用科大讯飞时最终单价为 0.45 元/分钟。
- 使用未配置专属价的其他供应商时，最终单价为 0.40 元/分钟。

### 6.3 单价快照

任务创建时，根据租户、模型和供应商解析最终单价，并保存为任务单价快照：

\[
\boxed{p_i=P(t_i,m_i,v_i)}
\]

任务创建后，后续调整模型默认价或供应商专属价，不影响该任务的冻结和结算。

## 7. 任务冻结

任务发起时，根据预计通话分钟数和单价快照计算冻结金额。

单个任务冻结金额：

\[
\boxed{F_i=m_i\times p_i}
\]

租户有效冻结总额：

\[
\boxed{F=\sum_{i=1}^{n}F_i}
\]

其中：

\[
p_i=P(t_i,m_i,v_i)
\]

第 \(i\) 个任务计入有效冻结总额，必须同时满足：

\[
\boxed{
\text{冻结状态}_i=\text{冻结中}
\land
\text{任务状态}_i\notin\{\text{已完成},\text{已终止}\}
\land
T_{\text{now}}-T_{\text{created},i}<24\text{小时}
}
\]

任务已完成、已终止或冻结创建满 24 小时时，立即解除冻结占用。释放记录需要保存释放时间和释放原因，并保证幂等。

即使后续修改模型默认价或供应商专属价，已有任务的冻结金额也不会改变。

## 8. 可用余额

可用余额计算公式：

\[
\boxed{A=\max(B-F,0)}
\]

即：

\[
\text{可用余额}
=
\max(
\text{资金余额}
-
\text{有效冻结总额},
0)
\]

可用余额只作为内部隐藏字段，用于：

- 换算大模型可用分钟数。
- 换算小模型可用分钟数。
- 校验本次导入所需冻结金额。

## 9. 本次导入冻结校验

设：

- \(N\)：本次导入号码数。
- \(Q\)：单号码预计通话分钟数。
- \(P(t,m,v)\)：租户、模型、供应商解析出的有效单价。
- \(A\)：扣除已有有效冻结后的可用金额。

本次导入需要新增冻结的金额为：

\[
\boxed{F_{\text{request}}=N\times Q\times P(t,m,v)}
\]

允许导入的条件为：

\[
\boxed{F_{\text{request}}\le A}
\]

最大可导入号码数为：

\[
\boxed{
N_{\max}
=
\left\lfloor
\frac{A}{Q\times P(t,m,v)}
\right\rfloor
}
\]

处理规则：

- 仍有效的已有冻结任务保持不变。
- 已完成、已终止或超过 24 小时的冻结任务先自动释放，再计算可用金额。
- 本次导入数不超过 \(N_{\max}\) 时允许导入，并新增对应冻结。
- 本次导入数超过 \(N_{\max}\) 时拒绝导入，并提示最大可导入数。
- 当 \(B<F\) 时，按既有公式得到 \(A=0\)，因此当前最大可导入数自然为 0；不再额外设置“冻结覆盖不足”的独立全局拦截状态。

## 10. 模型与供应商分钟数换算

### 10.1 当前模型默认价视图

当前版本页面按大模型默认价和小模型默认价展示分钟数。

#### 通话余额分钟数

大模型通话余额：

\[
\boxed{M_{LB}=\frac{\max(B,0)}{P_L}}
\]

小模型通话余额：

\[
\boxed{M_{SB}=\frac{\max(B,0)}{P_S}}
\]

#### 可用分钟数

大模型可用分钟数：

\[
\boxed{M_{LA}=\frac{A}{P_L}}
\]

小模型可用分钟数：

\[
\boxed{M_{SA}=\frac{A}{P_S}}
\]

大小模型分钟数是同一金额池的两种换算视图：

\[
M_{LA}+M_{SA}
\]

没有业务含义，禁止相加。

### 10.2 供应商专属价视图

启用供应商专属价后，指定租户、模型和供应商的通话余额分钟数为：

\[
\boxed{
M_B(t,m,v)=
\frac{\max(B,0)}{P(t,m,v)}
}
\]

指定租户、模型和供应商的可用分钟数为：

\[
\boxed{
M_A(t,m,v)=
\frac{A}{P(t,m,v)}
}
\]

例如，同一租户的大模型：

- 科大讯飞单价为 0.45 元/分钟。
- 一知科技单价为 0.40 元/分钟。
- 当前可用余额为 1000 元。

则：

\[
M_A(t,\text{大模型},\text{科大讯飞})
=
\frac{1000}{0.45}
\approx2222.22\text{ 分钟}
\]

\[
M_A(t,\text{大模型},\text{一知科技})
=
\frac{1000}{0.40}
=2500\text{ 分钟}
\]

这些分钟数都是同一可用余额的不同换算视图，并不是多个独立余额。因此：

- 不同模型的分钟数不能相加。
- 同一模型下不同供应商的分钟数不能相加。
- 不同模型、不同供应商的分钟数不能相加。
- 实际冻结和扣费必须使用任务选定供应商对应的单价快照。

## 11. 实际通话结算

任务完成时，先根据实际通话分钟数和任务单价快照计算实际消费金额，再立即解除该任务的冻结占用。任务终止或冻结创建满 24 小时时，也立即解除冻结占用。

第 \(i\) 个任务的实际消费金额：

\[
\boxed{C_i=m_{a,i}\times p_i}
\]

其中：

\[
p_i=P(t_i,m_i,v_i)
\]

任务原冻结金额：

\[
F_i=m_i\times p_i
\]

未使用的冻结金额：

\[
\boxed{U_i=\max(F_i-C_i,0)}
\]

任务结算后：

\[
B'=B-C_i
\]

\[
F'=F-F_i
\]

\[
A'=\max(B'-F',0)
\]

说明：释放未使用冻结金额不是给资金余额增加一笔收入，而是从冻结占用中移除该任务的原冻结金额。

### 11.1 自动释放条件

设冻结创建时间为 \(T_c\)，当前时间为 \(T_n\)，任务状态为 \(S\)。自动释放条件为：

\[
\boxed{
S\in\{\text{已完成},\text{已终止}\}
\quad\lor\quad
T_n-T_c\ge24\text{小时}
}
\]

释放后该任务不再计入有效冻结总额：

\[
F_i'=0
\]

资金余额 \(B\) 不因释放动作直接变化；任务产生的实际消费由结算动作单独扣减。

## 12. 呼叫条件

租户允许发起新呼叫任务，需要同时满足：

\[
\boxed{
\text{租户在有效期内}
\land
A>0
\land
\text{手动呼叫控制已启用}
}
\]

创建具体任务时，还必须能够解析出该任务的有效单价：

\[
\boxed{P(t,m,v)>0}
\]

若供应商没有专属价，则必须存在启用的模型默认价；如果两类价格均不存在或已停用，则该模型和供应商组合不得发起任务。

出现以下任一情况时，禁止发起新呼叫：

- 租户不在有效期内。
- 手动呼叫控制已停用。
- 无法解析出有效单价。
- 本次预计冻结金额大于可用金额。

## 13. 综合公式

当前可用金额可以归纳为：

\[
\boxed{
A=
\max(
R-D-C-F,
0)
}
\]

即：

\[
\boxed{
\text{可用金额}
=
\max(
\text{已生效通话充值}
-
\text{手工扣减}
-
\text{实际消费}
-
\text{有效冻结金额},
0)
}
\]

## 14. 计算示例

假设某租户：

- 已生效通话充值总额：\(R=2000\) 元
- 累计手工扣减：\(D=100\) 元
- 累计实际消费：\(C=180\) 元
- 有效冻结总额：\(F=720\) 元
- 大模型单价：\(P_L=0.40\) 元/分钟
- 小模型单价：\(P_S=0.26\) 元/分钟

资金余额：

\[
B=2000-100-180=1720
\]

可用余额：

\[
A=\max(1720-720,0)=1000
\]

大模型通话余额：

\[
M_{LB}=\frac{1720}{0.40}=4300\text{ 分钟}
\]

大模型可用分钟数：

\[
M_{LA}=\frac{1000}{0.40}=2500\text{ 分钟}
\]

小模型通话余额：

\[
M_{SB}=\frac{1720}{0.26}\approx6615.38\text{ 分钟}
\]

小模型可用分钟数：

\[
M_{SA}=\frac{1000}{0.26}\approx3846.15\text{ 分钟}
\]

如果再手工扣减 \(1100\) 元：

\[
B'=1720-1100=620
\]

由于：

\[
620<720
\]

因此可用金额为：

\[
A'=0
\]

大模型和小模型可用分钟数均为 0，本次最大可导入号码数也为 0。该结果来自请求级冻结校验，不再使用独立的“冻结覆盖不足”全局状态。

### 14.1 导入 2000 条与 5000 条示例

假设当前可用金额为：

\[
A=1000
\]

场景使用大模型，供应商为科大讯飞，未配置供应商专属价，因此回退模型默认价：

\[
P(t,m,v)=0.40
\]

单号码预计通话时长为：

\[
Q=1\text{ 分钟}
\]

最大可导入号码数：

\[
N_{\max}
=
\left\lfloor
\frac{1000}{1\times0.40}
\right\rfloor
=2500
\]

导入 2000 条时：

\[
F_{\text{request}}=2000\times1\times0.40=800\le1000
\]

允许导入。

导入 5000 条时：

\[
F_{\text{request}}=5000\times1\times0.40=2000>1000
\]

拒绝导入，并提示最多可导入 2500 条。
