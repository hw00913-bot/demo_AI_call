# 租户计费原型需求基线

## 页面范围

在现有租户管理列表中增加计费信息和充值管理入口，不改变其他业务模块。

## 租户列表

展示租户的计费配置入口、大模型可用分钟数、小模型可用分钟数、有效期和呼叫控制状态。点击“计费配置”按钮只维护该租户的大模型默认单价、小模型默认单价和启停状态。

## 充值管理

- 充值管理抽屉分为“计费明细”“充值单管理”和“余额调整”三个 Tab，默认展示计费明细。
- 计费明细 Tab 按大模型、小模型展示默认单价、冻结分钟数、冻结金额、通话余额和可用分钟数。
- 充值单管理 Tab 包含充值单关联、充值预览、统一资金说明和历史关联充值单。
- 余额调整 Tab 仅支持不关联充值单的手工扣减，并展示不可删除的扣减流水。
- 充值单关联成功后仅进入“待生效”状态，不立即修改租户有效期或统一资金余额。
- 每条已关联充值单独立提供“生效”操作；仅坐席费单生效后更新有效期，仅通话费单生效后更新余额，组合单生效后同时更新有效期和余额。
- 已生效充值单不可重复生效，历史列表展示支付状态、生效状态、生效有效期和操作。
- 外部充值单只包含充值金额，不包含模型类型。
- 所有通话充值进入统一资金账户；大模型和小模型不形成独立资金账户。
- 当前价格配置唯一键为租户、模型类型，供应商字段不参与当前版本计价。
- 计费配置弹窗支持修改大模型、小模型默认分钟单价和启停状态，单价必须大于 0。
- 数据结构保留可选供应商覆盖规则；未来启用时按“供应商专属价优先，模型默认价兜底”解析。
- 冻结任务锁定模型类型、供应商、冻结分钟数和任务单价快照，冻结金额统一从资金余额中占用。
- 扣除所有冻结任务金额后得到可用金额，再按大模型、小模型默认单价换算可用分钟数。
- 线下业务处理完成后，可通过手工扣减同步调整租户统一资金余额，不关联具体充值单。
- 手工扣减必须填写金额和原因，不提供手工补回、线上退款、撤销或修改。
- 手工扣减金额只允许不超过当前资金余额，不以“资金余额减冻结金额”作为扣减上限。
- 扣减后资金余额不足覆盖冻结金额时，不调整已有冻结任务；统一可用金额继续按最低 0 计算。
- 手动导入按“号码数 × 单号码预计分钟 × 有效单价”计算本次预计冻结金额，不超过当前可用金额时允许导入。
- 最大可导入号码数按“可用金额 ÷（单号码预计分钟 × 有效单价）”向下取整，并在超限时提示。
- 可用余额仅作为内部隐藏字段，不在页面以金额展示；页面展示大模型、小模型对应分钟数。
- 通过充值单号输入框提交校验。
- 支持仅坐席费、仅通话费、坐席费加通话费三类数据。
- 仅已支付且未重复关联的充值单进入历史列表。
- 所有已支付单据关联后均需逐单手动生效，再按计费类型更新有效期和余额。
- 历史列表仅展示已支付单据，不提供删除操作。

## 状态与异常

保留不存在、查询失败、已取消等异常结果供当前提交结果展示，但不写入历史关联记录。过期、统一可用金额为 0 或手动停用时，租户不可发起呼叫且管理按钮置灰；具体导入请求还需通过本次预计冻结金额校验。

## 非本期范围

真实接口、后端账务、并发冻结、最终扣费、金额释放和异常补偿不在原型中实现，通过流程图说明。
