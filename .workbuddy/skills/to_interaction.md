# Skill: 生成 CSV 说明文档 (Markdown 转换为 CSV)

**说明**：  
此 AI Skill 用于将结构化的 Markdown 功能说明文档（如 `功能说明文档.md`）无损地转换为可以在 Excel 中打开查看的 CSV 格式。该转换会保留文档中原有的段落文本、项目列表以及 Markdown 表格格式，避免仅提取表格时丢失上下文文字的问题。

## 触发条件
当用户说“生成 CSV 说明文档”、“将 md 导出为 cvs/csv” 或类似需求时，触发此技能。同时，如果在未来会话中涉及**生成/更新功能说明文档**的任务，也应遵循以下文档结构规范。

## 规范约束：功能说明文档结构
当负责撰写或更新《功能说明文档》及其 CSV 导出件时，整体内容必须严格遵从以下 5 个固定结构：
1. **版本说明**：【注意】只有当用户明确要求生成此项时才创建。若创建，必须使用**表格方式**显示，且表头严格为：`版本号` | `更新时间` | `更新内容`。
2. **数据来源说明**：必须使用**表格方式**显示，说明该功能模块所涉及数据的来源及底层接口流向。表头严格为：`系统名称` | `取值逻辑` | `取值字段`。
3. **逻辑说明**：必须使用**表格方式**显示，表头严格为：`流程名称` | `跳转地址`。用于描述功能的关联流程及链接跳转。
4. **功能说明**：作为一级标题“四. 功能说明”。内部必须严格遵从以下多级递增标号的结构（若有多个板块和功能，须依次生成标号）：
   - 二级标题：`1. 板块名称`（例：1. 板块A、2. 板块B）
   - 三级标题：
     - `1）功能名称`
     - `2）交互说明`
     - `3）数据来源`
     - `4）逻辑说明`
     - `5）功能明细`：此部分必须使用**表格方式**显示，将详细说明（如各字段描述）按列表结构排布。
5. **其他说明**：作为一级标题“五. 其他说明”。用于放置如“状态枚举映射表”等全局、通用的表格及说明内容。

## 执行步骤

1. **直接生成 CSV**：无需事先生成或更新 Markdown (.md) 格式的文件。AI 需在明确功能规范后，直接将结果写入到 CSV 文件中。
2. **生成方式**：
   - 建议 AI 使用 `write_to_file` 直接按 CSV 格式规范（逗号分隔，必要时用双引号包裹含换行或逗号的文本）生成文件。
   - 或者使用 Python 脚本生成 `csv`，若用脚本务必指定 `encoding='utf-8-sig'` 以防止 Excel 中文乱码。
3. **排版格式**：
   - 表格内容（数据来源说明等）正常多列分布。
   - 纯文本或层级说明（四. 功能说明等）可以直接置于第一列进行分行展示。
4. **默认输出**：目标路径默认为 `csv_exports/功能说明.csv`。生成完成后通知用户即可。
